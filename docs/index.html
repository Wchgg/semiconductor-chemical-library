Semiconductor Emergency Command Center

半导体应急指挥系统

面向晶圆厂与洁净生产区的厂级应急指挥界面，围绕应变处置、事件管理、营运恢复三段逻辑，整合 ALOHA、GMS、跨厂 CCTV、点名、SOP 与营运恢复判断。
事故类型：化灾 / 火灾 / 气灾 / 地震 / 漏水 / 台风 / 暴雨
核心模块：ALOHA / GHS / GMS / 点名 / CCTV / 厂广播
适用场景：晶圆厂 / 洁净室 / 特气站 / 化学品区 / 厂务系统

应变处置
控制现场、切断危险源、疏散与点名闭环、启动监控联动。
实时动作
  • 厂广播
  • 火警联动
  • GMS 二次复测
  • 二次点名
事件管理
把现场救急转成跨部门统筹，持续推进任务、告警、支援和复盘。
跨部门协调
  • 跨厂 CCTV
  • 事故快报
  • WIP 冻结
  • 外部支援请求
营运恢复
围绕复产关卡、公辅恢复、洁净恢复度和批次放行做恢复判断。
复产验证
  • 环境放行
  • 设备保全
  • 批次追溯
  • 管理放行

指挥总览

统一展示当前厂区、事故类型、事故区域、值守总指挥、实时风况和风险级别。

当前事件已触发

滚动显示告警、GMS 读值、报警位置和最新指挥动作，并带最新标记。

功能性按键

广播、CCTV、ALOHA、WIP、火警联动、点名、厂务隔离、事故快报都已接入状态联动。

ALOHA 与 GHS

支持化学品选择、扩散快速推估、风场联动、GHS 信号词、PPE、禁忌和首要动作。

GMS 与监控接口

页面内展示点位、读值、报警阈值、报警位置，并预留 CCTV、火警报警、广播接口。

洁净室内外点名

区分洁净室内与室外集合点，支持 badge / 门禁 / 承包商 / 访客多口径复核。

页面结构

当前仓库中的 Python 应用包含以下页面：综合态势、应变处置、事件管理、营运恢复、ALOHA 工作台、预案与 SOP、厂区分区、通信日志。 由于 GitHub Pages 只能托管静态页面，这个在线页用于长期查看系统说明和代码入口；完整动态交互仍在 Streamlit 应用里。
